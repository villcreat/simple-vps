# Шаблон плагина для VPS Simple

Готовый каркас плагина. Скопируйте папку `plugin-template`, переименуйте её и
заполните своими значениями.

## Структура

```text
plugin-template/
  plugin.yaml              # манифест: id, name, version, summary, entry по ОС
  scripts/
    ubuntu_install.yaml    # установщик для Ubuntu (bash)
    debian_install.yaml    # установщик для Debian (bash)
    windows_install.yaml   # установщик для Windows Server (PowerShell)
  ui/
    card_schema.json       # как выглядит карточка установленного сервиса
  README.md                # этот файл
```

## Как заполнять

1. В `plugin.yaml` задайте уникальный `id`, `name`, `version` и пути `entry` к
   нужным установщикам.
2. В `scripts/<os>.yaml` опишите шаги установки. Поля шага:

   | Поле            | Назначение                                                        |
   |-----------------|-------------------------------------------------------------------|
   | `id`            | уникальный id шага (для логов и отката)                           |
   | `title`         | что показать пользователю                                          |
   | `command`       | команда (bash для Linux, PowerShell для Windows)                  |
   | `safe`          | `true`, если шаг только читает/проверяет, ничего не меняя         |
   | `dangerous`     | `true` для опасных действий (требуют подтверждения фразой)        |
   | `reason`        | пояснение, почему шаг опасен / что делает                         |
   | `files_changed` | список изменяемых файлов (для бэкапа и отката)                    |
   | `ports_opened`  | список открываемых портов (показывается в предпросмотре)          |
   | `rollback`      | команда отката для этого шага                                     |

3. (Необязательно) Настройте `ui/card_schema.json` — поля и кнопки карточки.

## Безопасность

- Любой шаг проходит через **dry-run → предпросмотр → подтверждение → лог**.
- Опасные команды (например `rm -rf`) **автоматически** помечаются опасными,
  даже если в YAML указано `dangerous: false`. Скрыть опасность нельзя.
- Всегда указывайте `rollback`, где это возможно — это включает кнопку отката.

## Как загрузить

- В приложении: экран **Плагины → указать папку плагина → выбрать ОС**.
- В коде/тестах: `PluginLoader.loadFromDirectory('<путь>', ServerOs.ubuntu)`.

Плагин загружается на время сессии (в память) и запускается через тот же
безопасный поток, что и встроенный каталог сервисов.
